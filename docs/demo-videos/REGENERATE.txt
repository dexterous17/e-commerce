Regenerate demo MP4 files (Playwright captures real browser automation).

Requirements:
  - API healthy and at least one product with countInStock > 0
  - ffmpeg on PATH (e.g. brew install ffmpeg)
  - For local Vite stack: Postgres + DATABASE_URL for the backend; from frontend/, `npm run test:e2e:demo-videos`

Pacing (optional):
  Recordings default to slowMo=125ms between actions and longer “breathe” pauses.
  Faster local/CI: DEMO_SLOW_MO=0 npm run test:e2e:demo-videos

Cursor / tooling note:
  Some environments pin PLAYWRIGHT_BROWSERS_PATH to an empty sandbox path. Unset it before install + test:
    env -u PLAYWRIGHT_BROWSERS_PATH npx playwright install chromium

How files get here:
  Playwright saves each clip as frontend/test-results/<run>/video.webm (WebM only).
  Successful `npm run test:e2e:demo-videos*` runs transcode into this folder via
  `frontend/scripts/sync_demo_mp4_from_results.py`. To sync manually after a run:
    cd frontend && npm run sync-demo-mp4

Local recordings:

  cd frontend && env -u PLAYWRIGHT_BROWSERS_PATH npm run test:e2e:demo-videos

Against production https://ecommerce.harshildex.com (creates real disposable users / test orders):

  WARNING: Hits your live storefront and APIs. Prefer off-peak runs.

    cd frontend && npm run test:e2e:demo-videos:prod

  Equivalent manual env:

    env -u PLAYWRIGHT_BROWSERS_PATH DEMO_RECORD_RUN_ID=$(date +%s) \\
      PLAYWRIGHT_SKIP_WEBSERVER=1 PLAYWRIGHT_BASE_URL=https://ecommerce.harshildex.com \\
      E2E_API_ORIGIN=https://ecommerce.harshildex.com npm run test:e2e:demo-videos

Against an already-running dev server on 5173:

  PW_REUSE_SERVER=1 PLAYWRIGHT_SKIP_WEBSERVER=1 env -u PLAYWRIGHT_BROWSERS_PATH npm run test:e2e:demo-videos

Against Docker Compose static frontend on 3000:

  PLAYWRIGHT_SKIP_WEBSERVER=1 PLAYWRIGHT_BASE_URL=http://127.0.0.1:3000 E2E_API_ORIGIN=http://127.0.0.1:5004 env -u PLAYWRIGHT_BROWSERS_PATH npm run test:e2e:demo-videos

Outputs (MP4 in this folder):
  01-storefront-tour.mp4 — home, browse, product detail, add to cart
  02-register-session.mp4 — shopper registration UI
  03-checkout-flow.mp4 — sign in, checkout through order confirmation
  04-search-catalog.mp4 — search header, filtered list, open a product
  05-sign-in-session.mp4 — API seed user then sign-in UI
  06-profile-orders.mp4 — sign in, profile and My Orders
