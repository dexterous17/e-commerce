Nginx Proxy Manager (NPM) — route harshildex.com to this project
================================================================

If ecommerce.harshildex.com shows a *different* site, NPM default page, or 404:
  1) DNS must point to THIS server only:  dig +short ecommerce.harshildex.com
  2) In NPM (port 81), open Proxy Hosts — ensure ecommerce.harshildex.com forwards to
     frontend:80 (Option A) or host:8080 (Option B), not another container or old IP.
  3) Delete duplicate Proxy Host rows for the same domain; avoid a catch-all *.harshildex.com
     that steals traffic unless it forwards to this app.
  4) Re-apply automation (creates missing hosts OR fixes wrong upstream):

       ./scripts/npm-repair-proxy-hosts.sh

     (from repo root on the server; needs NPM admin password in env — see env/npm-bootstrap/.env.example)

Expected HTML marker for this storefront: <title>Tailored by Harshil</title>
  curl -sS https://ecommerce.harshildex.com/ | head -20

================================================================

OPTION A (recommended) — NPM in Docker next to this repo’s stack
----------------------------------------------------------------
From the repo root:

  docker compose -f docker-compose.lightsail.yml -f docker-compose.nginx-proxy-manager.yml up -d --build

The npm-bootstrap service creates proxy hosts, can fix an existing host that points at
the wrong upstream, and can attach a shared Let's Encrypt certificate when
NPM_LETSENCRYPT_EMAIL (or a real NPM_ADMIN_EMAIL) is configured
(re-run with ./scripts/npm-repair-proxy-hosts.sh).

Open the admin UI on port 81 (see docker-compose.nginx-proxy-manager.yml header).

Manual Proxy Hosts (if bootstrap is disabled) — container hostnames:

  ecommerce.harshildex.com
    Scheme: http
    Forward hostname: frontend
    Forward port: 80

  backend.ecommerce.harshildex.com
    Scheme: http
    Forward hostname: backend
    Forward port: 5000

SSL tab: only needed if you skip bootstrap TLS. When bootstrap TLS is enabled, the script
requests the certificate and attaches it for you; use NPM_FORCE_SSL=1 for HTTPS redirects.

Advanced tab — paste Snippet A for the storefront host, Snippet B for the API host (below).

--------------------------------------------------------------------
OPTION B — NPM already runs elsewhere (separate Docker stack / VM)
--------------------------------------------------------------------
Your browser hits NPM first. NPM must “Proxy Host” each domain to the
Docker ports published by docker-compose.lightsail.yml (defaults:
frontend 127.0.0.1:8080, API 127.0.0.1:5004).

Important: NPM runs inside a container. 127.0.0.1 there is the NPM
container, not your host. Use one of these as “Forward Hostname / IP”:

  1) host.docker.internal   (best if your NPM stack defines it; common on Docker Desktop)
  2) 172.17.0.1              (often the host on the default docker bridge — try if (1) fails)
  3) Your server’s LAN/private IP (last resort)

Forward scheme: HTTP (NPM terminates HTTPS; upstream stays HTTP on loopback).

Proxy Host 1 — storefront (SPA from frontend; Snippet A can send /api, /api/media, /uploads to API)

  Domain names:     ecommerce.harshildex.com
  Scheme:           http
  Forward hostname: host.docker.internal   (or 172.17.0.1)
  Forward port:     8080
  (Optional) Block Common Exploits: ON
  (Optional) Websockets Support: ON

Proxy Host 2 — API hostname (optional)

  Domain names:     backend.ecommerce.harshildex.com
  Scheme:           http
  Forward hostname: host.docker.internal   (or 172.17.0.1)
  Forward port:     5004

================================================================
Snippet A — storefront proxy host
================================================================

client_max_body_size 50m;
proxy_read_timeout 300s;
proxy_send_timeout 300s;

# Route /api (REST), S3 media proxy, and /uploads to the API (same upstream as
# backend.ecommerce.harshildex.com). List /api/media before /api so longer prefix wins.
# Option A (NPM on Docker network with this stack): use backend:5000.
# Option B (NPM → host loopback): use 127.0.0.1:5004 or your BACKEND_PORT.

location ^~ /api/media {
  proxy_pass http://backend:5000;
  proxy_http_version 1.1;
  proxy_set_header Host $host;
  proxy_set_header X-Real-IP $remote_addr;
  proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
  proxy_set_header X-Forwarded-Proto $scheme;
  proxy_buffering off;
  proxy_request_buffering off;
  proxy_read_timeout 300s;
}

location ^~ /api {
  proxy_pass http://backend:5000;
  proxy_http_version 1.1;
  proxy_set_header Host $host;
  proxy_set_header X-Real-IP $remote_addr;
  proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
  proxy_set_header X-Forwarded-Proto $scheme;
  proxy_buffering off;
  proxy_request_buffering off;
  proxy_read_timeout 300s;
}

location ^~ /uploads {
  proxy_pass http://backend:5000;
  proxy_http_version 1.1;
  proxy_set_header Host $host;
  proxy_set_header X-Real-IP $remote_addr;
  proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
  proxy_set_header X-Forwarded-Proto $scheme;
  proxy_read_timeout 300s;
}

Verify on the host (Option B / published ports)
================================================================

  curl -sS http://127.0.0.1:8080/ | head
  curl -sS http://127.0.0.1:5004/api/health

If those work but NPM does not, the Forward hostname for NPM is wrong.

================================================================

After deploy — catch broken API / images early
================================================================

From the repo root (any machine that can reach your public URL):

  ./scripts/verify-deployment-endpoints.sh https://ecommerce.harshildex.com

This checks /api/health, featured products JSON, and an HTTP HEAD on the first
product image (S3 proxy or uploads). If it fails, fix proxy hosts and env:

  ./scripts/npm-repair-proxy-hosts.sh

Ensure env/aws/.env on the server has credentials (or IAM role) so
/api/media/s3 can call S3; see /api/health field s3ImageProxy.
